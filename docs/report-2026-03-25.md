4. Fix: Notifikasi "Mode Offline" Padahal Online

4.1 Root Cause Analysis
Tiga masalah berlapis menyebabkan app selalu menampilkan "Tersimpan secara Lokal":

1. hasInternetAccess() pakai HttpClient (dart:io) - berbeda HTTP stack dengan Dio yang dipakai API call. HEAD request gagal meskipun server reachable via Dio.
2. AttendanceService.checkIn() membungkus DioException menjadi Exception - tipe error hilang, _isNetworkError() tidak bisa bedakan network vs server error.
3. Semua exception ditangkap sebagai network error - server reject (422/403/500) tetap masuk path offline.

4.2 Fixes

File: connectivity_service.dart
Perubahan: Ganti HttpClient.headUrl() menjadi Dio.get('auth/me') dengan validateStatus: (_) => true. Server reachable jika response < 500 (bahkan 401 = reachable)

File: attendance_service.dart
Perubahan: DioException di-rethrow langsung, tidak dibungkus Exception

File: offline_attendance_handler.dart
Perubahan: Pisah catch: on DioException (cek e.type) vs generic catch. badResponse = server error (tampil ke user), connectionTimeout/connectionError = network (simpan offline)


5. Fix: Face Verification Tidak Perlu untuk Operational Track

5.1 Problem
AttendanceController.store() memaksa face verification untuk semua absensi. Karyawan operational track (outlet) yang belum daftar wajah diblokir dengan error 403.

5.2 Fix

File: AttendanceController.php
Perubahan: Terima parameter track_type. Skip face verification & geofence validation jika track_type = 'operational'

File: AttendanceService.dart
Perubahan: Tambah parameter trackType pada checkIn()

File: offline_attendance_handler.dart
Perubahan: Pass trackType ke checkIn()

Aturan baru:
- Operational track: QR scan + wide selfie = bukti kehadiran (tanpa face verification & geofence)
- Head office track: GPS + face verification (tetap seperti sebelumnya)


6. QR Code Format Baru

6.1 Problem
QR code menggunakan format OUTLET-32-LT1-1773720000-A3B2 yang tidak readable.

6.2 Fix

File: QrCodeValidationService.php
Perubahan: generateQrCode() sekarang menggunakan code dari tabel organizations. Format baru: {ORG_CODE}-LT{FLOOR}-{RANDOM} (contoh: KINGTECH-T3F-CGK-LT1-A3B2). Fallback ke format lama jika org belum punya code.

File: offline_attendance_handler.dart
Perubahan: _isValidQrFormat() diupdate untuk menerima format baru dan legacy

Catatan: QR lama yang sudah aktif tetap valid (validasi by exact match di database). QR baru yang di-generate akan menggunakan format baru.
